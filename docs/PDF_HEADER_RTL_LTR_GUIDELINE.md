# IMS PDF Header RTL/LTR Guideline

**Author:** Manus AI | **Date:** March 6, 2026 | **Version:** 1.0

---

## 1. Purpose

This document establishes the **mandatory standard** for implementing PDF document headers across all IMS modules. Every PDF generated by the system — whether in Logistics, Finance, HR, MEAL, Projects, or any future module — must follow this guideline to ensure consistent, correct RTL (Arabic) and LTR (English) header mirroring.

The standard is based on the **BOM (Bid Opening Minutes) implementation**, which has been validated in production and confirmed by the user as the correct reference.

---

## 2. Expected Header Layout

The official PDF header consists of three columns arranged horizontally across the page width. The positions of the left and right columns **swap automatically** when the language direction changes, while the center column remains fixed.

| Direction | Physical LEFT (28%) | Physical CENTER (44%) | Physical RIGHT (28%) |
|-----------|--------------------|-----------------------|---------------------|
| **LTR (English)** | Org Name, OU Name, Department | Document Title | Logo, Ref Number, Date |
| **RTL (Arabic)** | Logo, Ref Number, Date | Document Title | Org Name, OU Name, Department |

The center column always contains the document title. The key behavior is that the **organization identity** and the **logo + reference** columns swap sides when switching between LTR and RTL. This mirroring happens automatically through CSS flexbox — no separate HTML templates or conditional branching is needed.

---

## 3. The Proven Approach: Flexbox via `official-pdf.css`

The only approved CSS file for PDF headers is:

```
server/services/pdf/templates/styles/official-pdf.css
```

This file uses **CSS flexbox** (`display: flex; justify-content: space-between;`) on the `.official-header` container. When the `<html>` element has `dir="rtl"`, the browser's flexbox engine automatically reverses the visual order of flex children — placing the first child (`.header-left`) on the physical right side and the last child (`.header-right`) on the physical left side. No JavaScript, no conditional HTML, no grid-template-areas swap is needed.

### Why Flexbox Works

Flexbox respects the CSS `direction` property inherited from `<html dir="rtl">`. When `direction: rtl` is active, `justify-content: space-between` distributes items from right to left instead of left to right. This is the W3C-specified behavior and works reliably in Puppeteer's Chromium rendering engine.

### Why CSS Grid Failed

The previous `official-pdf-v2.css` used CSS Grid with `grid-template-areas` to swap column positions in RTL. While this approach is valid in theory, it **did not work reliably in Puppeteer's rendering engine**, resulting in headers that remained in LTR layout even when `dir="rtl"` was set. This approach is now deprecated and must not be used for PDF headers.

## 4. Reference Implementation: `OfficialWrapper.ts`

The canonical implementation lives in:

```
server/services/pdf/templates/layout/OfficialWrapper.ts
```

This file exports `generateOfficialPdfHtml(options)` which produces a complete HTML document with the correct header structure. Below is the exact header HTML pattern that every PDF must follow:

```html
<!DOCTYPE html>
<html dir="${direction}" lang="${language}">
<head>
  <meta charset="UTF-8" />
  <style>${officialCss}</style>  <!-- official-pdf.css content -->
</head>
<body>

  <div class="official-header">
    <div class="header-left">
      <div class="org-block">
        <div class="org-name">${organizationName}</div>
        <div class="ou-name">${operatingUnitName}</div>
        <div class="module-name">${department}</div>
      </div>
    </div>

    <div class="header-center">
      <div class="doc-title">${documentTitle}</div>
    </div>

    <div class="header-right">
      <img class="org-logo" src="${logoUrl}" alt="Logo" />
      <div class="ref-date">
        <div class="value ltr-safe">${refNumber}</div>
        <div class="value ltr-safe">${date}</div>
      </div>
    </div>
  </div>
  <hr class="hr-strong" />

  <div class="pdf-content">
    ${bodyHtml}
  </div>

</body>
</html>
```

### Critical Rules in This Pattern

The following table summarizes the mandatory rules that must be observed when implementing the header:

| Rule | Description |
|------|-------------|
| **Single HTML template** | There must be ONE header template for both LTR and RTL. Never create separate HTML branches like `if (isRTL) { ... } else { ... }` for the header. |
| **`<html dir="${direction}">`** | The `dir` attribute on the `<html>` element is the ONLY mechanism that controls RTL/LTR. Set it to `"rtl"` for Arabic and `"ltr"` for English. |
| **Load `official-pdf.css`** | Always load the CSS from `server/services/pdf/templates/styles/official-pdf.css`. Never use `official-pdf-v2.css` or inline CSS Grid rules for the header. |
| **Use `.official-header`** | The parent container class must be `official-header` (not `pdf-header`). This class has `display: flex; justify-content: space-between;`. |
| **Child order: left, center, right** | The HTML order must always be `.header-left` first, `.header-center` second, `.header-right` third. CSS handles the visual reordering in RTL. |
| **`.ltr-safe` on references** | Reference numbers, dates, and codes must have the class `ltr-safe` to prevent Arabic numeral reordering. This class applies `direction: ltr; unicode-bidi: isolate;`. |
| **Org name stays in English** | The organization name remains in English even in Arabic mode. Only the department name, document title, and labels are translated. |

## 5. How to Load the CSS

Every PDF generator that uses Puppeteer must load the CSS file by reading it from disk and injecting it into a `<style>` tag. The path is always relative to the file's location using `import.meta.dirname`:

```typescript
import fs from "fs";
import path from "path";

// Load the official PDF CSS (BOM's flexbox approach)
const cssPath = path.join(
  import.meta.dirname,
  "../services/pdf/templates/styles/official-pdf.css"
);
const css = fs.readFileSync(cssPath, "utf-8");
```

If the PDF generator is located in `server/_core/`, the relative path would be:

```typescript
const cssPath = path.join(
  import.meta.dirname,
  "../services/pdf/templates/styles/official-pdf.css"
);
```

The CSS content is then injected into the HTML `<style>` tag. Additional inline styles for the body content (tables, sections, etc.) can be added in a separate `<style>` block after the official CSS, but **must never override the `.official-header` flexbox rules**.

---

## 6. Inline Style Overrides: What Is Allowed

You may add inline `<style>` blocks for body-specific styling (table widths, section colors, page breaks, etc.), but the following rules apply:

| Allowed | Not Allowed |
|---------|-------------|
| Table column widths, borders, colors | Overriding `.official-header` display property |
| Section title font sizes | Changing `justify-content` on `.official-header` |
| Page break rules (`page-break-before`) | Adding `grid-template-areas` to `.official-header` |
| Body font size adjustments | Changing `.header-left`, `.header-center`, `.header-right` widths |
| RTL overrides for table `th`/`td` alignment | Using `float` or `position: absolute` on header elements |

If you need to adjust the document title font size within the header, use an inline `style` attribute on the `.doc-title` element itself, not a CSS rule that targets the class:

```html
<div class="doc-title" style="font-size:14pt;">${title}</div>
```

## 7. Checklist for New PDF Documents

When creating a new PDF generator for any module, follow this checklist:

| Step | Action | Verification |
|------|--------|-------------|
| 1 | Set `<html dir="${direction}" lang="${language}">` | Check that `direction` is `"rtl"` for Arabic, `"ltr"` for English |
| 2 | Load `official-pdf.css` from `server/services/pdf/templates/styles/` | Verify the path resolves correctly from your file location |
| 3 | Use `<div class="official-header">` as the header container | Confirm it is NOT `pdf-header` or any other class name |
| 4 | Place org info in `.header-left > .org-block` | Org name, OU name, department in that order |
| 5 | Place title in `.header-center > .doc-title` | Centered, bold, translated |
| 6 | Place logo + ref in `.header-right` | Logo image, then `.ref-date` with `.ltr-safe` values |
| 7 | Add `<hr class="hr-strong" />` after the header | Visual separator between header and content |
| 8 | Wrap body content in `<div class="pdf-content">` | Ensures proper width and direction inheritance |
| 9 | Test in English (LTR) | Org LEFT, Title CENTER, Logo+Ref RIGHT |
| 10 | Test in Arabic (RTL) | Logo+Ref LEFT, Title CENTER, Org RIGHT |

## 8. Common Mistakes to Avoid

**Mistake 1: Dual HTML branches for RTL/LTR headers.**
Creating separate HTML blocks like `if (isRTL) { headerRTL } else { headerLTR }` defeats the purpose of CSS-based mirroring and introduces maintenance burden. The flexbox approach handles this automatically.

**Mistake 2: Using `official-pdf-v2.css` with CSS Grid.**
The `grid-template-areas` swap approach does not work reliably in Puppeteer. Always use `official-pdf.css` with flexbox.

**Mistake 3: Hardcoding `text-align: left` or `text-align: right` on header elements.**
Use `text-align: start` and `text-align: end` instead. These logical properties respect the document direction automatically.

**Mistake 4: Forgetting `.ltr-safe` on reference numbers and dates.**
Without this class, Arabic bidirectional text rendering can reorder digits and hyphens in reference numbers like `PR-EFADAH01-2026-031`, making them unreadable.

**Mistake 5: Overriding header CSS with inline styles.**
Adding `style="display: grid"` or `style="float: left"` to header elements will break the flexbox mirroring. Only override font sizes and colors, never layout properties.

## 10. Testing Protocol

Every PDF must be tested in both languages before deployment. Use the following URL patterns to test directly in the browser:

```
# English (LTR)
/organization/logistics/bid-analysis/{id}/evaluation-checklist-pdf?lang=en&orgId={orgId}
/organization/logistics/bid-analysis/{id}/cba-pdf?lang=en&orgId={orgId}

# Arabic (RTL)
/organization/logistics/bid-analysis/{id}/evaluation-checklist-pdf?lang=ar&orgId={orgId}
/organization/logistics/bid-analysis/{id}/cba-pdf?lang=ar&orgId={orgId}
```

For each test, verify the header visually against this reference:

| Check | LTR Expected | RTL Expected |
|-------|-------------|-------------|
| Physical LEFT column | Org name, OU, Department | Logo (centered), Ref number, Date |
| Physical CENTER column | Document title (bold, centered) | Document title (bold, centered) |
| Physical RIGHT column | Logo (centered), Ref number, Date | Org name, OU, Department |
| Separator line | Full-width dark line below header | Full-width dark line below header |
| Ref number direction | Left-to-right digits | Left-to-right digits (`.ltr-safe`) |

## 12. Summary

The golden rule is simple: **use `official-pdf.css` with the `.official-header` flexbox container, set `dir="rtl"` or `dir="ltr"` on the `<html>` element, and write a single HTML template.** The CSS handles everything else. Never branch the header HTML for different languages, never use CSS Grid for the header, and always test in both English and Arabic before shipping.
